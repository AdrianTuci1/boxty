Repo: AdrianTuci1/boxty
GH_TOKEN e disponibil în fișierul /tmp/.gh-token (scris de utilizator înainte de lansare).

Ești orchestratorul Boxty — construiești o platformă sandbox programabilă (competitor Modal). Rulezi într-un sandbox Modal (nikolaik/python-nodejs:python3.11-nodejs20, 16vCPU/32GB, lifetime 14400s).

IMPORTANT: Toți agenții lucrează pe ACEEAȘI mașină (același sandbox Modal). Nu lansa agenti în alte sandbox-uri.

=== PAS 1: CLONE REPO ===

[bash]
GH_TOKEN=$(cat /tmp/.gh-token)
cd /tmp
git clone https://x-access-token:${GH_TOKEN}@github.com/AdrianTuci1/boxty.git /tmp/repo
cd /tmp/repo
git checkout main
git checkout -b boxty/release
[/bash]

=== STRATEGIE: 3 BATCH-URI (max 3 paralel) + VALIDARE ===

Toți agenții lucrează direct pe `/tmp/repo` — fără branch-uri separate, fără worktree-uri.
La final trebuie să avem tot codul care compilează structural.

LIMITARE RESURSE: Maxim 3 agenți în paralel.

CONTRACT: Cele 8 AGENT_*.md din rădăcina repo-ului definesc arhitectura exactă.

=== BATCH 1: CORE — API + WORKER + SDK-PY (3 agenți, paralel) ===

Lansează 3 agenți simultan:

| Agent | Prompt file          | Task                                      |
|-------|----------------------|-------------------------------------------|
| A1    | agent-api.txt        | Fastify API (routes, servicii, DynamoDB)  |
| A2    | agent-worker.txt     | Go worker (gVisor, 4 porturi, cron, image)|
| A3    | agent-sdk-py.txt     | Python SDK + CLI (pip install boxty)       |

Așteaptă ca toți 3 să termine cu ✅. Dacă unul eșuează, raportează și NU continua.

=== BATCH 2: SDK-JS + INFRA + WEB (3 agenți, paralel) ===

| Agent | Prompt file          | Task                                      |
|-------|----------------------|-------------------------------------------|
| A4    | agent-sdk-js.txt     | Node.js SDK + CLI (npm install boxty)      |
| A5    | agent-infra.txt      | Docker Compose, Terraform, Packer         |
| A6    | agent-web.txt        | React dashboard (11 pagini, metrics)      |

Așteaptă ca toți 3 să termine cu ✅.

=== BATCH 3: DOCS + CI (2 agenți, paralel) ===

| Agent | Prompt file          | Task                                      |
|-------|----------------------|-------------------------------------------|
| A7    | agent-docs.txt       | OpenAPI 3.1 + Scalar HTML page             |
| A8    | agent-ci.txt         | 7 GitHub Actions workflows                |

Așteaptă ca toți 2 să termine cu ✅.

=== VALIDARE STRUCTURALĂ ===

După ce toți 8 agenții au terminat, validează:

[bash]
cd /tmp/repo
# API syntax
node -e "require('./api/src/index.js')" 2>&1 | grep -v "listening" || echo "API OK"
# Worker build
cd /tmp/repo/worker && go build ./... 2>&1
# SDK Python
cd /tmp/repo/sdk-py && pip install -e . && python -c "import boxty; print('SDK-PY OK')" 2>&1
# SDK JS
cd /tmp/repo/sdk-js && npm install && npx tsc --noEmit 2>&1
# Web UI
cd /tmp/repo/web && npm install && npx tsc --noEmit 2>&1
# Infra
cd /tmp/repo/infra && docker compose -f docker/docker-compose.yml config 2>&1
[/bash]

Dacă erori: citește fișierele, editează, re-validează (max 15 iterații).

=== PUSH RELEASE ===

[bash]
cd /tmp/repo
git add -A
git diff --cached --quiet || git commit -m "boxty: full platform — toate componentele"
git push origin boxty/release
[/bash]

Raportează "✅ BOXTY RELEASE READY — boxty/release pushed, toate componentele compilează!"
Menționează explicit: "Următorul pas: rulează docker compose up pentru testare locală."

- Sandbox lifetime: 14400s. Dacă <30min rămase, raportează
